# Multi-Container Application Deployment on AWS

## Overview
This project automates the deployment of a two-tier web application (Flask + MySQL) on an AWS EC2 instance. The infrastructure is provisioned using **Terraform**, configured via **Ansible**, and orchestrated using **Docker Compose**.

The architecture follows a modular DevOps workflow, ensuring scalability, security, and reproducibility.

## Architecture
* **Infrastructure**: AWS EC2 (**t3.micro**) running Amazon Linux 2.
* **Provisioning**: Terraform with a remote S3 backend and DynamoDB state locking.
* **Configuration**: Ansible for automated environment setup (Docker, Docker Compose, Buildx).
* **Orchestration**: Docker Compose managing a Python Flask frontend and a MySQL database backend.

## Prerequisites
* Terraform (v1.5.0+)
* Ansible (v2.10+)
* AWS CLI configured with appropriate credentials.

## Deployment Steps

### 1. Infrastructure Provisioning (Terraform)
Navigate to the `terraform/` directory.

1.  **Initialize local state**:
    ```bash
    terraform init
    ```
2.  **Deploy infrastructure**:
    ```bash
    terraform apply -auto-approve
    ```
3.  **Configure Remote Backend**:
    Copy the `state_bucket_name` from the outputs. Uncomment the `backend "s3"` block in `main.tf`, paste the bucket name, and run:
    ```bash
    terraform init
    ```
    Confirm **"yes"** to migrate your state to S3.

### 2. Application Deployment (Ansible)
Navigate to the `ansible/` directory. The `inventory.ini` and SSH `.pem` key have been automatically generated in the previous step.

1.  **Run the playbook**:
    ```bash
    ansible-playbook playbook.yml
    ```

## Verification
Once the deployment is complete, verify the application is running by accessing the EC2 public IP on port 5000:

```bash
curl http://<EC2_PUBLIC_IP>:5000
```

## Security & Best Practices
* **State Locking**: Prevents concurrent state changes using DynamoDB.
* **Least Privilege**: Security groups are restricted to essential ports (22 for SSH, 5000 for App).
* **Automated Cleanup**: Use the following to tear down the environment and avoid AWS costs:
    ```bash
    terraform destroy -auto-approve
    ```
